Toast 1 - JavaScript 自定義錯誤 Error

May 2, 2022 · 3 min read
Callum
Front End Engineer

前言​

平常 Coding 過程中，會使用流程控制陳述式 / 條件陳述式去撰寫一項功能這是很常見的。 但是某些時候輸入的值，是無法掌握的、是外部的沒辦法預期結果，只能驗證各方面錯誤的可能。

舉例來說撰寫 POST 行為時，給於既定規範的前提下要驗證使用者是否按規則走。

function listener(req, res) {
    let data = '';
    req.on('data', (chunk) => {
        data += chunk;
    });

    if (req.method === 'POST') {
        req.on('end', () => {
            try {
                const obj = JSON.parse(data);
                // 1. 驗證 data 是否為 JSON String?
                // 2. 驗證 obj.title 是否有值?
            } catch (error) {
                if (error instanceof SyntaxError) {
                    // JSON parse error
                }
            }
        });
    }
}

當 POST 後我們處理好 buffer 問題，預設規範上寫 data 是 JSON String 要轉為 Object 但是發生格式不對 JavaScript 會拋 SyntaxError 例外錯誤 (Javascript 語法錯誤的 Error)

問題是驗證 obj.title 是否能有一個自定義的錯誤，提供給程式流程判斷呢 🤔

自定義錯誤​

JavaScript 對於 Error 有最上層的物件名為 Error，可以使用 Class 繼承它來客製化

class CustomizeError extends Error {
    constructor(message) {
        super(message);
        this.name = this.constructor.name;
    }
}

透過 super 存取父層把 message 存進去，接著我們把上面例子拿過來

function listener(req, res) {
    let data = '';
    req.on('data', (chunk) => {
        data += chunk;
    });

    if (req.method === 'POST') {
        req.on('end', () => {
            try {
                const obj = JSON.parse(data);
                if (!obj?.title) {
                    throw new CustomizeError('title 為必填');
                }
            } catch (error) {
                if (error instanceof SyntaxError) {
                    console.log('JSON parse error');
                }
                if (error instanceof CustomizeError) {
                    console.log(error.message);
                }
            }
        });
    }
}
  • JSON.parse(data) 當語法錯誤 JavaScript 引擎會拋例外錯誤 SyntaxError
  • !obj?.title 當驗證沒有 title 屬性，就會拋例外錯誤 CustomizeError
  • 進入 catch 使用 instanceof 判斷，可以客製化各 Error 物件後續的處理
