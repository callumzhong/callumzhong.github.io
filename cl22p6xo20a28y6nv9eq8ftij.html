Mock API 身為前端該了解的知識點

April 17, 2022 · 5 min read
Callum
Front End Engineer

簡介​

Mock 詞意為模擬的意思，那麼先了解為什麼需要模擬?

我們所開發的網頁應用程式大部分搭配資料庫，較大型的網頁則可能會有多個複雜的資料庫， 這需要後端開發人員的協作。

但是開發需要時間去完成，為了讓整體開發流程順利，則模擬 API 回傳數據會是好的解決方案。另外網頁有使用第三方 API 的話，利用 MOCK API 於開發期間也能降低調用第三方 API 的成本。

選擇工具​

從簡介了解到 MOCK API 意思後該怎麼建立呢 ?

推薦使用 Mockoon 服務

  • 不需要遠端部屬
  • 不必建立帳戶
  • 免費
  • 開源

點擊 Download 選擇你的作業系統下載即可

官網上的畫面介紹圖

安裝好後讓我們開始建立第一個 Mock API

使用 GUI 建立 MOCK API​

打開 mockoon 會看到官方範例 API ，我從這邊以圖示上的編號介紹

  1. environment 是 Mock API 環境檔 (儲存在本地端的 json)
  2. Route 是路由以 Mock 環境為目錄可建立多個 API
  3. Body 是 Response Body
  4. Headers 是該 Route 的 Headers
  5. Headers 是該 Mock API 環境檔的 Headers (進去有按鈕可以加入 CORS 跨域)

其實畫面非常直觀沒什麼好講的，但是湊字數再加點圖示

  • 新增 environment

  • 新增路由

到此停一下，我們加入特別來賓 SWAPI API 來模擬它

如何運行​

  • 可以同時把多個環境檔運行起來，但要小心 PORT 不要衝突 (mockoon 有防呆機制)
  • 設定 PORT 的方式在上方 Settings > API URL 標題那個區塊

Route Rules​

你可以對 API 建立 Requset 規則。 假設模擬 Login API 指定某個使用者帳號才可登入

  • 定義 Header Content-Type 為 application/json
  • 定義 body 屬性有 username 值是 example@gmail.com
  • 定義 body 屬性有 password 值是 1234

設定完後記得再加一個 Response 為 401 順序要替換一下預設是 401
